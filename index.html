Primavera

Welcome to the 2020 Coding Contest!

Primavera is a spring coding contest in which users participate all throughout Spring in coding challenges.

Individual winners may receive some awards, but bigger events may call for bigger rewards. At the end of Spring the judges will pick one or more users who showed amazing talent.

Challenges could be themed or open to you. All of that will be revealed during the course of the contest.

Start date: March 16-22, 2020

Fun Fact: Primavera means spring in Spanish

Joining

person_add

Why join?

So you can participate in the most awesome coding contest on Khan Academy.

How to join

Joining is easy; simply post a comment in the JOIN THREAD in the Tips & Thanks telling which team you prefer to be in and how advanced of a programmer you are (0-4):

  • 0 - Beginner
  • 1 - Familiar
  • 2 - Intermediate
  • 3 - Advanced
  • 4 - Expert

After that, post a SINGLE period . in the NEWS THREAD so you'll get notified when the new challenges come out.

NOTE: If you wish to judge in this contest instead of participating, that is also fine. Up to 3-4 other users besides myself have the option to help judge.

Interested in judging?

Ask in the Tips & Thanks if you think you are a capable judge. This will require that you are at least an intermediate level programmer.

When we judge

Each challenge will last 1-3 weeks; at the end of the challenge the Judges will need to judge the teams' projects. As soon as all the judges' results get in I will look them over (and possibly tweak them a bit) and post the final results at the News thread. When judging, a program rated 5.5/10 will be rated "average". Don't feel bad if you get less than the average though, that just means you have room to learn! Take the opportunity and do all you can to learn from your programming!

What we judge on

Judges will vote based upon the following algorithm;

  • Fitting theme -> If the user completely ignores the theme they will score less points.
  • Correct Code/No Logic Errors -> If judges see programs with "dead code" or code that is not working correctly they will score less points.
  • Code readability/format (comments, indention, ect) -> If the program has messy code that is hard to read it will score less points.
  • Efficiency (lag free, minimal repeated code, no unecessary intensive tasks) -> The more effecient the program's code is the better it scores!
  • Graphic Design/Creativity -> Creative Programs or ones that show excelent graphic design will score more.
  • Complexity -> Similar to efficiency, if you have particularly advanced code that reduces effort and time you wills score more.
  • Visual Design (how the program presents itself) -> In some ways similar to Graphic Design, but includes other things besides just how good the graphics are.

Scoring and results

We divide all the contestants into their level group; Each level group is as follows (0, 1, 2, 3, 4) which makes a total of 5 groups. Each user will score X/10 points where 5.5/10 is the average program. Don't feel bad if you didn't score well, you just need to work harder ad learn more for the next challenge!

The best program from each group will "win" that challenge and be shown in the NEWS THREAD as well as the user who made that being awarded 5 points.

At the end of spring the team with the most points (from all their members) will be declared the winning team.

There will also be "5 Amazing Programmers" featured for their demonstrated skill and hard work.
